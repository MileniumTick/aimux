# Spec: Release Pipeline

## Requirement

Automated cross-platform release builds triggered by Git tags, published as GitHub Releases with downloadable archives and a Homebrew formula.

## Scope

- `.goreleaser.yml` configuration file at the project root
- `.github/workflows/release.yml` GitHub Actions workflow
- No Windows builds (out of scope per proposal)
- Homebrew formula published to `jchavarriam/homebrew-tap`

## Implementation

### `.goreleaser.yml`

```yaml
version: 2
project_name: aimux

before:
  hooks:
    - go mod tidy

builds:
  - env:
      - CGO_ENABLED=0
    goos:
      - darwin
      - linux
    goarch:
      - amd64
      - arm64
    mod_timestamp: "{{ .CommitTimestamp }}"
    flags:
      - -trimpath
    ldflags:
      - -s -w
      - -X main.Version={{.Version}}
      - -X main.Commit={{.FullCommit}}
      - -X main.Date={{.Date}}

archives:
  - format: tar.gz
    name_template: >-
      {{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}
    files:
      - LICENSE
      - README.md

checksum:
  name_template: "checksums.txt"

brews:
  - name: aimux
    repository:
      owner: jchavarriam
      name: homebrew-tap
      branch: main
    homepage: "https://github.com/jchavarriam/aimux"
    description: "AI Multiplexer — switch AI models across CLIs from a single TUI"
    license: "MIT"
    test: |
      system "#{bin}/aimux --version"
```

Key configuration details:

- `CGO_ENABLED=0`: produces fully static binaries (safe because `modernc.org/sqlite` is pure Go)
- `-trimpath`: produces reproducible builds without local filesystem paths
- `-s -w`: strips debug symbols for smaller binary size
- 4 platform targets: `darwin/amd64`, `darwin/arm64`, `linux/amd64`, `linux/arm64`
- Archive name format: `aimux_1.2.0_darwin_arm64.tar.gz`
- Homebrew formula auto-generated by GoReleaser with correct SHA256 checksum

### `.github/workflows/release.yml`

```yaml
name: Release

on:
  push:
    tags:
      - "v*"

jobs:
  goreleaser:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: "1.23"

      - name: Run GoReleaser
        uses: goreleaser/goreleaser-action@v6
        with:
          distribution: goreleaser
          version: "~> v2"
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

Key workflow details:

- Trigger: push of any tag matching `v*` (e.g., `v0.2.0`, `v1.0.0`)
- Runs on `ubuntu-latest` (cross-compiles all platforms from a single Linux runner)
- `fetch-depth: 0` required by GoReleaser for commit timestamp and changelog generation
- `GITHUB_TOKEN` is automatically provided by GitHub Actions (no PAT needed for publishing to the same repo)
- Homebrew formula push to `jchavarriam/homebrew-tap` requires `GITHUB_TOKEN` scoped to that repo; if `secrets.GITHUB_TOKEN` is insufficient (cross-repo push), a PAT must be configured as `HOMEBREW_TAP_TOKEN` and used for the brew step

### Homebrew Tap Repository

The tap repository at `github.com/jchavarriam/homebrew-tap` must exist before the first release. It should contain:

- A `Formula/` directory
- An initial empty commit or README

GoReleaser will create/update `Formula/aimux.rb` on each release with:
- The correct version
- The correct SHA256 checksum for the darwin_amd64 archive
- The download URL pointing to the GitHub Release

Users install via:
```
brew install jchavarriam/tap/aimux
```

## Scenarios

### S1: `git tag v1.0.0` triggers GitHub Actions workflow

**Given** a push of tag `v1.0.0` to the main branch
**When** GitHub Actions processes the push event
**Then** the release workflow triggers
**And** creates a GitHub Release with tag `v1.0.0`

### S2: GoReleaser builds 4 platforms

**Given** the release workflow runs
**When** GoReleaser builds
**Then** 4 archives are produced:
- `aimux_1.0.0_darwin_amd64.tar.gz`
- `aimux_1.0.0_darwin_arm64.tar.gz`
- `aimux_1.0.0_linux_amd64.tar.gz`
- `aimux_1.0.0_linux_arm64.tar.gz`
**And** each archive is a valid `tar.gz` containing the binary, LICENSE, and README.md

### S3: CGO_ENABLED=0, static binaries with -trimpath

**Given** a release build
**When** the binary is inspected with `file`
**Then** it shows `statically linked`
**When** the binary is inspected with `go tool buildid`
**Then** no local filesystem paths appear in the binary

### S4: Generates checksums.txt

**Given** a release build completes
**When** GoReleaser finishes
**Then** a `checksums.txt` file is included in the release artifacts
**And** it contains SHA256 checksums for all 4 archive files

### S5: Publishes GitHub Release with tarballs

**Given** GoReleaser completes successfully
**When** the release is published
**Then** the GitHub Release page shows 4 archive downloads and `checksums.txt`
**And** the release is tagged with the exact tag name (e.g., `v1.0.0`)

### S6: Publishes Homebrew formula to jchavarriam/homebrew-tap

**Given** GoReleaser publishes a release
**When** the brew step runs
**Then** a commit is pushed to `jchavarriam/homebrew-tap` updating `Formula/aimux.rb`
**And** the formula contains the correct version, SHA256, and download URL

### S7: Formula uses `brew install jchavarriam/tap/aimux`

**Given** the Homebrew formula is published
**When** a user runs `brew install jchavarriam/tap/aimux`
**Then** the binary is installed at the Homebrew prefix
**And** `aimux --version` works and prints the installed version

### S8: Non-tag pushes do not trigger release

**Given** a push to `main` without a `v*` tag
**When** GitHub Actions processes the push event
**Then** the release workflow does NOT run
